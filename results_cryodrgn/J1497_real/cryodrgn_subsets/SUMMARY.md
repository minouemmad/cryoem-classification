# cryoDRGN J1497 particle subsets

Confidence-tiered `.cs` files for CryoSPARC refinement, derived from the J1497 z10
cryoDRGN run (`results_cryodrgn/J1497_real/train/z.100.pkl`, 230,396 particles).
Generated by `scripts/cryodrgn/export_cryodrgn_subsets.py`.

Note: The main finding is that P9 and P10 are structurally indistinguishable from P8
and P6 respectively (LDA recall P9=0.016, P10=0.010). The hc-tier counts for P9/P10
are very small (see below), reflecting high assignment ambiguity for those classes.

## Files in this folder

| File | Tier | Description |
|------|------|-------------|
| `cryodrgn_class_P{6,7,8,9,10}.cs` | full | All particles with latent-GMM argmax = class |
| `cryodrgn_class_P{6,7,8,9,10}_hard.cs` | hard | latent-GMM argmax == CryoSPARC argmax |
| `cryodrgn_class_P{6,7,8,9,10}_hcperm.cs` | hcperm | hard AND JS < 66th percentile |
| `cryodrgn_class_P{6,7,8,9,10}_hc.cs` | hc | hard AND JS < 33rd percentile |
| `cryodrgn_posteriors.csv` | — | Per-particle uid, cryoDRGN posterior (P6-P10), JS divergence, agreement flag |

Total protein-only: 230,396. Hard agreement: 56.1% (vs 81.4% for J1442 K=3),
confirming the 5-class partition is harder for cryoDRGN to resolve.

## Injected field

Every `.cs` file has a `cryodrgn/class_posterior` field (shape N x 5, float32, rows
sum to 1.0) injected at export time alongside all original CryoSPARC pose/CTF fields.

## JS divergence thresholds

JS(CryoSPARC posterior, cryoDRGN posterior) per particle:
- 33rd percentile: hc threshold
- 66th percentile: hcperm threshold
- Mean = 0.348 nats, median = 0.372 nats (higher than J1442 due to poorer 5-class agreement)

## Recommended use

- P6, P7, P8 subsets are usable for refinement (hard tier has 18k-59k particles).
- P9 and P10 hard-tier counts (17k and 1k) are too small and structurally ambiguous
  to yield reliable unique reconstructions; P9 particles reconstruct as P8, P10 as P6.
- If pursuing the 5-class set, use the J1442 3-class subsets (`J1442_real/cryodrgn_subsets/`)
  instead, which have better separation and more balanced particle counts.
- **CryoSPARC workflow:** Import Particles -> Ab-initio (K=1, initial_res=12) -> NU-Refinement.
