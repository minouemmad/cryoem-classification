# cryoDRGN J1442 particle subsets

Confidence-tiered `.cs` files for CryoSPARC refinement, derived from the J1442 z10
cryoDRGN run (`results_cryodrgn/J1442_real/train_z10/z.100.pkl`, 230,396 particles).
Generated by `scripts/cryodrgn/export_cryodrgn_subsets.py`.

## Files in this folder

| File | Tier | Description |
|------|------|-------------|
| `cryodrgn_class_P{6,7,8}.cs` | full | All particles with latent-GMM argmax = class |
| `cryodrgn_class_P{6,7,8}_hard.cs` | hard | latent-GMM argmax == CryoSPARC argmax |
| `cryodrgn_class_P{6,7,8}_hcperm.cs` | hcperm | hard AND JS < 66th percentile (0.381 nats) |
| `cryodrgn_class_P{6,7,8}_hc.cs` | hc | hard AND JS < 33rd percentile (0.294 nats) |
| `cryodrgn_posteriors.csv` | — | Per-particle uid, cryoDRGN posterior (P6/P7/P8), JS divergence, agreement flag |

Total protein-only: 230,396. Hard agreement: 81.4% overall.

## Injected field

Every `.cs` file has a `cryodrgn/class_posterior` field (shape N x 3, float32, rows
sum to 1.0) injected at export time alongside all original CryoSPARC pose/CTF fields.

## JS divergence thresholds

JS(CryoSPARC posterior, cryoDRGN posterior) per particle:
- 33rd percentile = 0.294 nats  -> `hc` threshold
- 66th percentile = 0.381 nats  -> `hcperm` threshold
- Mean = 0.298 nats, median = 0.299 nats, bound = ln(2) = 0.693 nats

## Recommended use

- **Tier selection guide:**
  - `full`: maximum particle count; use for initial latent-GMM-based refinement.
  - `hard`: removes cross-method disagreement (~18% of full); best general starting point.
  - `hcperm`: ~65% of hard; lower uncertainty; reasonable sample size per class.
  - `hc`: ~35% of hard; purest signal per class; smallest N, expect degraded FSC for P7 (15k particles).

- **CryoSPARC workflow:** Import Particles -> Ab-initio (K=1, initial_res=12) -> NU-Refinement.
